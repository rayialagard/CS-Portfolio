# CS 210 Portfolio Artifact

## Project Summary

For this project, I created a C++ program for Corner Grocer that reads grocery purchase data from a file and counts how many times each item was purchased. The program allows the user to search for an item's frequency, display all item frequencies, and print a histogram. This helps the store better understand customer purchasing patterns.

## What I Did Well

I think I did a good job organizing my program by using a class and separate functions for each task. This made the code easier to read, test, and maintain. I also used a map to efficiently store and retrieve grocery item frequencies.

## Areas for Improvement

If I continued working on this project, I would improve the input validation and error handling. I would also make the search feature ignore uppercase and lowercase letters so users could search more easily. These improvements would make the program more reliable and user-friendly.

## Challenges and Support

The most challenging part of this project was working with file input and output while using a map to correctly count each grocery item. I overcame this by testing my program often, reviewing the course resources, and using the C++ documentation to better understand how maps and file handling work.

## Transferable Skills

This project strengthened my skills in C++, object-oriented programming, file input and output, functions, classes, and problem-solving. These skills will be useful in future programming courses and software development projects.

## Maintainability, Readability, and Adaptability

I made the program maintainable by organizing the code into a class with separate functions for each task. I used meaningful variable names and comments to improve readability. This structure also makes it easier to update the program or add new features in the future without changing the entire code.
