# CS 210 Portfolio Artifact

## Project Summary

For this project, I created a C++ program for Corner Grocer that reads a grocery purchase file and counts how many times each item was purchased. The program allows the user to search for a specific item, display all item frequencies, and print a histogram. The goal of the program is to help the store better understand customer buying habits.

## What I Did Well

I think I did a good job organizing my code by using a class and separate functions for each task. This made the program easier to read and helped keep each part of the code focused on one job.

## Areas for Improvement

If I were to improve this project, I would add stronger error handling and make the menu more user-friendly. I could also make the program ignore uppercase and lowercase differences when searching for items. These changes would make the program easier to use and more reliable.

## Challenges and Support

The most challenging part of this project was working with file input and using a map to count each grocery item correctly. I overcame this by testing my program often, reviewing the course materials, and using the C++ documentation when I got stuck.

## Transferable Skills

This project strengthened my skills in C++, file input and output, classes, functions, maps, and problem-solving. These are skills I will continue to use in future programming courses and software development projects.

## Maintainability, Readability, and Adaptability

I made my program easy to read by using meaningful variable names, comments, and separating the code into functions. Organizing the program this way makes it easier to update, fix, or add new features in the future without changing the entire program.
